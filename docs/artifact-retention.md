# Artifact retention policy

Generated run output grows with every fleet run and season. This is the policy
for what belongs in the repository and what does not, so it doesn't compound
unbounded. See also [`.gitignore`](../.gitignore), which enforces the "raw
output" exclusions below.

## Tiers

### 1. Canonical input — committed forever
The authoritative, hand-authored source of the competition. Small, reviewed,
and versioned.
- `schemas/`, `archive/schemas/` (retired-but-loadable v1 schemas)
- `tasks/`, `rounds/`, `rubrics/`, `oracle/` (private keys; see #256)
- `fixtures/`, `adapters/`, `scripts/`, `test/`, `docs/`

### 2. Scored record — committed, curated
The minimal set needed to reproduce the public leaderboard and to preserve
season history. Committed deliberately, not as a dumping ground.
- `results/*.yaml` — per-participant scored result packets and judge records
  that the scoreboard is built from.
- `archive/season-*/…` — dated, superseded run bundles kept for history (see
  [`archive/README.md`](../archive/README.md)).
- `evidence/dry-run/execute/…` — the reference dry-run capture used by docs and
  gates.

### 3. Raw run output — NOT committed
Transient, regenerable, or per-run artifacts. Excluded via `.gitignore`; store
out-of-repo (GitHub Releases / an archive branch) if long-term retention is
needed.
- `runs/` (entire tree) — live-runner and round-engine run directories.
- `results/scoreboard.json`, `results/*-auto-judge.yaml` — regenerated by
  `make score`.
- `results/perf-harness-report-*.{json,yaml}`, `results/pkt-harness-*.yaml` —
  regenerated by the perf harness.
- `evidence/dry-run/finalizer-evidence.json` and other regenerated gate
  evidence JSON.
- `.tmp/`, `web-output/`, `public-site/`.

## Drift guard

Committed **scored records** must not silently drift from what the scripts
produce. CI runs the generate steps (`make score`, `make dry-run-finalizer`)
and then `git diff --exit-code`, so a committed artifact that no longer matches
regeneration fails the build. If you intend to update a scored record,
regenerate and commit it in the same change.

## Adding a new artifact

Before committing generated output, place it in a tier:
- Reproducible from committed input by a script → **raw output**, add to
  `.gitignore`.
- Needed for the public leaderboard or season history → **scored record**,
  commit it and keep the drift guard green.
- Neither → it probably doesn't belong in the repo.
