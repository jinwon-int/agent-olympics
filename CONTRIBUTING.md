# Contributing

Agent Olympics contributions should keep the benchmark platform-neutral.

## Task Contributions

Good tasks include:

- A clear objective.
- Explicit allowed and forbidden actions.
- Required outputs.
- Evidence or fixture data sufficient for judging.
- Hidden judge notes or an answer key.

Do not include secrets, private logs, or irreversible production instructions.

## Reference Contributions

Reference issues should explain:

- What the external project is.
- What design lesson applies to Agent Olympics.
- Whether it affects tasks, adapters, judging, schemas, or leaderboards.

## Result Contributions

Result Packets should be redacted and schema-valid. They should preserve enough evidence for judging without leaking credentials or private data.
